# Product

## Register

product

## Users

Argus serves a single macOS developer working across terminals, source files,
git worktrees, and agent-driven tasks. Users need dense, keyboard-friendly
workspace management that preserves project and task context.

## Product Purpose

Argus keeps terminal, file, browser, and source-control work inside one native
workspace. Success means users can inspect and operate on project state without
losing their active workspace, tab, pane, or terminal focus.

## Brand Personality

Native, focused, precise. Interface should feel familiar to users of macOS
terminal applications, editors, and source-control clients, and should recede
behind current task.

## Anti-references

Avoid decorative dashboards, web-style card layouts, heavy gradients, novelty
controls, gratuitous animation, and content that opens in independent utility
windows. Do not trade desktop density or standard macOS behavior for visual
spectacle.

## Design Principles

- Keep work in its originating Workspace.
- Make interaction and current state legible before activation.
- Preserve selection, focus, and layout while content changes.
- Prefer native macOS controls and behavior over invented affordances.
- Use semantic status and color only where they communicate task state.

## Accessibility & Inclusion

Use accessible labels and help for icon-only controls, full and stable hit
areas, keyboard-equivalent actions, selectable content, adaptive light and dark
appearance, and non-color state indicators. Respect reduced motion and standard
macOS accessibility behavior.
